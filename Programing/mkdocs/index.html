    系統化探索 MkDocs × Mermaid 心得全紀錄¶

    MkDocs 常用指令大彙整 — 從入門到部署全流程

    魔鬼藏在細節裡。一段段的指令操作，看似簡單，如果失了順序或沒用對分類，排查起來可是又長又亂。下面直接按照自己的新手到實戰歷程，將這些關鍵步驟都系統化整理成一份超實用的工具表。

    部署¶

    🧰 安裝與初始化¶

    步驟 指令 說明
    1 pip install mkdocs 安裝 MkDocs
    2 mkdocs new my-project 建立新專案
    3 cd my-project 進入專案目錄
    4 ls 確認已建立 mkdocs.yml 與 docs/

    🛠️ 開發與預覽¶

    步驟 指令 說明
    1 mkdocs serve 啟動本地預覽伺服器 http://127.0.0.1:8000
    2 mkdocs build 編譯成靜態網站（輸出至 site/）
    3 mkdocs gh-deploy 發佈至 GitHub Pages（需設定 Git）

    ⚙️ 設定與擴充¶

    步驟 指令 說明
    1 編輯 mkdocs.yml 設定網站標題、主題、導覽列等
    2 pip install mkdocs-material 安裝 Material 主題
    3 pip install mkdocs-mermaid2-plugin 安裝 Mermaid 外掛
    4 在 mkdocs.yml 中啟用外掛 plugins: - mermaid2

    🧪 除錯與維護¶

    步驟 指令 說明
    1 mkdocs doctor 檢查設定檔與文件結構是否有誤
    2 mkdocs --help 查看所有可用指令與參數
    3 mkdocs build --clean 清除舊的 site/ 再重新編譯

    系統化探索概論¶

    類別 核心重點
    問題描述 Mermaid 文字顏色無法自訂，與主題、classDef、CSS 三方權重有衝突
    學習心法 模組化拆解問題、A/B 測試、善用可視化與紀錄專案演進

    Mermaid 文字顏色控制策略¶

    方案 設定方法 適用情境
    統一字色 .mermaid .label { color: inherit !important; } 統一整站主題文字色，風格一致
    強制自訂色 .mermaid text { fill: #xxxxxx !important; } 全站強制自訂字色，SVG 層級制霸
    完全自訂 plugin theme: null 搭自家 CSS 完全自由，全部用品牌風格設計

    問題核心歸納¶

    層級 行為與影響
    Mermaid 主題 依據 plugin 設定決定渲染優先權
    classDef Mermaid 內部樣式，有時候被主題與 CSS 蓋掉
    外部 CSS MkDocs、Material、extra.css 會高權重覆蓋

    補充建議清單：

    • 優先掌握外層 CSS 的覆蓋行為（善用 devtools 查 SVG 結構）

    • 進行 theme: base 與 theme: null A/B 測試尋找最適合自己的顏色流派

    • 必要時調高 CSS specificity，並善用 !important 調優

    • 新版 Mermaid 頻繁升級，遇問題時多測幾個圖類

    Trouble-shouting¶

    核心流程表¶

    問題類型／現象 對應解決策略 實戰補充建議
    文字顏色被主題鎖死（classDef 無效或被覆蓋） .mermaid .label { color: inherit !important; }（統一主題字色）
    .mermaid text { fill: #XXXXXX !important; }（暴力指定）
    plugin 設定 theme: null+自訂 CSS（完全自控）    		 用 devtools 查 SVG 權重
    對 theme: base/null AB 測試
    必要時提高 CSS specificity
    Mermaid v11 結構異動導致覆蓋失效 調整 CSS selector，正確對應新版 SVG 標籤結構 升級後要多測各種圖表型態，及時修改 selector 以適應
    主題設定、內建與外部 CSS 權重競爭 根據需求選擇主題或外部 CSS 控制，強化 selector 權重 發現無法覆蓋，首先檢查主題、Mermaid、CSS 三者權重相互覆蓋
